# sattvah-web

The public marketing + SEO surface for Sattvah — an emotional-support app and
community. Read-only: anyone can browse community posts and expert profiles.
Posting, replying and booking sessions deep-link into the mobile app via the
`sattvah://` universal link.

This is a **standalone** codebase. It only talks to the production Sattvah API
over HTTPS. No shared code with the mobile app or admin console.

## Stack

- **Next.js 14** (App Router, React Server Components)
- **Tailwind CSS** + shadcn/ui primitives (Button, Card, Badge, Avatar)
- **TypeScript strict**
- No client state library — server components + URL search params only
- ISR caching via `next: { revalidate: 60 | 120 }` on every backend fetch
- SEO: per-page `<title>` / `<meta description>`, OpenGraph, Twitter cards,
  JSON-LD (`Article` for posts, `Person` for experts), dynamic `sitemap.xml`
  and `robots.txt`

## Getting started

```bash
cp .env.example .env.local
npm install
npm run dev          # http://localhost:3000
```

```bash
npm run lint
npm run build
npm run start
```

Requires Node **20+** (Next 14).

## Environment

| Variable                          | Purpose                                          |
| --------------------------------- | ------------------------------------------------ |
| `NEXT_PUBLIC_API_BASE_URL`        | Sattvah backend (defaults to prod API Gateway)    |
| `NEXT_PUBLIC_SITE_URL`            | Canonical URL — used in metadata + sitemap       |
| `NEXT_PUBLIC_IOS_APP_URL`         | App Store URL (fallback for deep links)          |
| `NEXT_PUBLIC_ANDROID_APP_URL`     | Play Store URL (fallback for deep links)         |

Defaults are sensible for local dev; production values go in Vercel.

## Routes

| Route                          | Notes                                     |
| ------------------------------ | ----------------------------------------- |
| `/`                            | Landing page                              |
| `/community`                   | Approved-post feed (server-rendered)      |
| `/community/posts/[id]`        | Post detail + `Article` JSON-LD           |
| `/experts`                     | Expert browse; `?category=` filter        |
| `/experts/[id]`                | Profile + `Person` JSON-LD + book CTA     |
| `/about`                       | About page                                |
| `/privacy`, `/terms`           | Legal summaries                           |
| `/robots.txt`, `/sitemap.xml`  | Generated by `app/robots.ts` + `sitemap.ts` |

## Architecture notes

- `src/lib/api.ts` is the only place that touches the backend.
- Anonymous community posts are rendered as the backend serves them
  (`authorName: "Anonymous"` for non-author viewers).
- 403 / 404 responses degrade to an "Open in the app" CTA instead of crashing.
- Deep links use `sattvah://<path>`; iOS/Android intercept when the app is
  installed, otherwise the user can use the store CTAs in the header.

## Deployment

Configured for Vercel — push to `main` and import. `vercel.json` is minimal
on purpose; no rewrites currently. Add legal-proxy rewrites later if we want
`/legal/*` to serve from the backend directly.

## Conventions

- Tailwind only — no CSS modules, no styled-components.
- shadcn/ui components live in `src/components/ui/`.
- Comments explain **why**, not **what**.
